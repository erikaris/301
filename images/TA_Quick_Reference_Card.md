# Tableau V Session - TA Quick Reference Card

## SESSION AT A GLANCE
**Duration:** 105 minutes | **Format:** Team Project | **Classes:** 3-4

| Task | Time | Duration | Key Action |
|------|------|----------|-----------|
| Task 1: Team Setup | 0-10 min | 10 min | Teams select PM, create team name, display post-it |
| Task 2: Client Consultation | 10-15 min | 5 min | Brief all PMs (one at a time) with case scenarios |
| Task 3: Solution Development | 15-75 min | 60 min | Teams build dashboards/prototypes |
| Task 4: Solution Refinement | 75-105 min | 30 min | Client feedback → make refinements → submit |

---

## TASK 1: TEAM SETUP (10 minutes)

**Objective:** Organize teams with a Project Manager

**Your Job:**
- [ ] Announce task 1
- [ ] Ensure no one is working alone (minimum group of 2)
- [ ] Verify PM is named and post-it is visible
- [ ] Collect list of all PMs and team names

**What Teams Do:**
1. Decide who is Project Manager
2. Decide on team name
3. Write PM name + team name on post-it
4. Stick post-it on desk (visible)

**Checklist:**
- ✓ All teams formed
- ✓ No isolated students
- ✓ All post-its visible
- ✓ PM list created

**Sample Post-it:**
```
Sarah Johnson
Data Dashers
```

---

## TASK 2: CLIENT CONSULTATION (5 minutes)

**Objective:** Brief each PM on their case study

**Your Job:**
- [ ] Call PMs one at a time (in order)
- [ ] Give each PM their briefing sheet
- [ ] Explain the problem, dataset, questions, and deliverable options
- [ ] Answer questions about their scenario
- [ ] Send PM back to brief their team

**PM Schedule (Stagger by ~2-3 minutes):**
- PM 1: _____ Team: _____
- PM 2: _____ Team: _____
- PM 3: _____ Team: _____
- PM 4: _____ Team: _____
- PM 5: _____ Team: _____
- PM 6: _____ Team: _____

**Key Information to Communicate:**
✓ What problem are they solving?
✓ What dataset do they have?
✓ What 3-4 questions should they answer?
✓ Who is their audience?
✓ Dashboard (£5,000) or Prototype (£1,000)?
✓ What data file to use?

**Checklist:**
- ✓ All PMs briefed
- ✓ PMs understand the scenario
- ✓ PMs know where to find data
- ✓ Teams ready to start development

---

## TASK 3: SOLUTION DEVELOPMENT (60 minutes)

**Objective:** Teams create dashboards/prototypes

**Your Job:**
- [ ] Monitor progress around the room
- [ ] Ask guiding questions
- [ ] Help with technical issues
- [ ] Keep time and announce milestones

**Timeline & Checkpoints:**
- **Min 0-15:** Data exploration, planning, sketching
  - Check: Do teams understand their data?
  - Q: "What patterns do you see?"
  
- **Min 15-45:** Building visualizations
  - Check: Are charts being created?
  - Q: "Why did you choose that chart type?"
  
- **Min 30:** CHECKPOINT #1
  - Announce: "30 minutes down, 30 to go!"
  - Quick check-in with each team
  
- **Min 45-60:** Polishing and refining
  - Check: Are labels, titles, legends added?
  - Q: "Does this answer your client's questions?"
  
- **Min 45:** CHECKPOINT #2
  - Announce: "15 minutes left for development!"
  - Teams should be finalizing visuals

**Helpful Questions to Ask:**
- "What question are you trying to answer?"
- "Who will use this dashboard?"
- "What would they do with this information?"
- "Can you simplify this?"
- "What's your most important insight?"
- "Is this the best way to show this data?"

**Common Issues & Quick Fixes:**

| Issue | Solution |
|-------|----------|
| Team frozen, unsure where to start | Guide: Load data → explore → sketch → build |
| Wrong visualization chosen | Suggest: "Try a bar chart instead of pie" |
| Chart is cluttered | Recommend: Filter data, reduce categories, simplify |
| Team disagreeing on approach | Facilitate: Quick vote, move forward, iterate |
| Technical problems (software won't open) | Alternatives: Use Excel, hand-sketch, Powerpoint |

**Checklist:**
- ✓ All teams building something
- ✓ Time checks announced at 30 min & 45 min
- ✓ Technical issues addressed quickly
- ✓ Teams on track for 2-3 main visualizations

---

## TASK 4: SOLUTION REFINEMENT (30 minutes)

**Objective:** Finalize and polish dashboard, then submit to Padlet

**Your Job:**
- [ ] Monitor teams making final improvements
- [ ] Answer questions about whether solution addresses brief
- [ ] Help with file saving/exporting
- [ ] Remind about Padlet submission deadline

**What Teams Do:**
1. Review Task 2 client brief
2. Verify their solution addresses the original questions
3. Polish visualizations (titles, labels, clarity)
4. Add professional touches (colors, legends, formatting)
5. Export/save final file
6. Submit to Padlet

**Time Checkpoints:**
- **Min 0-15:** Teams making refinements
  - Announce at 10 min: "20 minutes left!"
- **Min 15-25:** Teams finalizing
  - Announce at 25 min: "5 minutes to deadline!"
- **Min 25-30:** Submission window
  - Announce at 30 min: "Time to submit!"

**Key Questions for Teams:**
- "Does this answer your client's original questions?"
- "Are all your charts labeled clearly?"
- "Would the client understand what this shows?"
- "Is everything accurate and professional?"

**Common Refinement Issues & Quick Fixes:**

| Issue | Solution |
|-------|----------|
| Chart missing title | Add descriptive title to every visualization |
| Colors don't match | Apply consistent color scheme throughout |
| Numbers don't look right | Quick spot-check against data; fix if wrong |
| Too cluttered | Hide non-essential elements, simplify |
| Missing axis labels | Add clear labels to all chart axes |

**Submission Support:**
- [ ] Padlet link working
- [ ] Teams know how to submit
- [ ] Files being saved correctly
- [ ] Submission deadline is clear

## RED FLAGS TO WATCH FOR

🚩 **Team Issues:**
- Team member not participating → Assign a specific task
- Team member dominating → Suggest collaborative approach
- Team disagreement → Facilitate compromise, remind of time
- No progress → Break down task into smaller steps

🚩 **Technical Issues:**
- Software not opening → Suggest Excel, sketches, alternatives
- File corruption → Help recover or start fresh
- Printer/export problems → Offer to help or provide alternatives

🚩 **Conceptual Issues:**
- Wrong visualization type → Suggest alternatives
- Answering wrong question → Reference client brief
- Data misinterpretation → Help verify numbers
- Analysis unclear → Ask "So what?" and "Now what?"

🚩 **Time Issues:**
- Team too slow → Suggest cutting scope (2-3 visuals instead of 5)
- Team too fast → Encourage adding polish and insights
- Refinement taking too long → Focus on top 1-2 changes only

**Emergency Interventions:**
- If a team is completely lost: Work through first chart together as an example
- If someone is falling behind: Pair with another student for peer support
- If tech fails entirely: Pivot to hand-sketching and verbal explanation
